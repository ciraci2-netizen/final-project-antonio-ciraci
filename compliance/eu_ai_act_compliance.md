## 2. Classification Reasoning — Step by Step

### Step 1 — Is this system prohibited?
The EU AI Act prohibits AI systems that manipulate persons through subliminal techniques, exploit vulnerabilities, enable social scoring by public authorities, or conduct real-time biometric surveillance.

**Conclusion:** This system does none of the above. It analyses publicly available property listing data and generates text recommendations for individual hosts. ✅ Not prohibited.

### Step 2 — Is this a High-Risk system?
High-risk AI systems include those used in:
- Critical infrastructure
- Education and vocational training
- Employment and worker management
- Access to essential private and public services
- Law enforcement
- Migration and border control
- Administration of justice

**Conclusion:** This system operates in the short-term rental market — a commercial, non-critical domain. It does not make decisions about people's access to services, employment, credit, or rights. It generates text recommendations that a human host can choose to accept or ignore. ✅ Not high-risk.

### Step 3 — Is this a Limited Risk system?
Limited risk applies to AI systems that interact with humans or generate content that could be mistaken for human-created content, including:
- Chatbots and conversational AI
- AI-generated text, images, or other content
- Emotion recognition systems
- Biometric categorisation systems

**Conclusion:** This system uses GPT-4o-mini to generate listing descriptions and business insights — AI-generated text content that a host may publish on Airbnb. Under the EU AI Act, systems that generate text which users may publish or present to others must disclose that the content is AI-generated. ✅ Limited risk applies.

### Step 4 — Summary of Classification

| Criterion | Assessment |
|-----------|------------|
| Prohibited use case | No |
| High-risk domain | No |
| Generates AI content for publication | Yes |
| Interacts directly with end users | Yes (via Streamlit UI) |
| **Final classification** | **LIMITED RISK** |

## 3. Mandatory Requirements for Limited Risk Systems

### 3.1 Transparency Obligation
Under Article 50 of the EU AI Act, providers of AI systems that generate text must ensure that:
- Users are informed that the content is AI-generated
- The disclosure is clear and visible at the point of output

**How this system addresses it:**
- The Streamlit interface displays a clear label: *"This description was generated by AI. Review and edit before publishing."*
- The `poc_documentation.md` and `mvp_documentation.md` both state explicitly that outputs are AI-generated
- LangSmith logs every generation event with metadata confirming AI authorship

### 3.2 Human Oversight
The system is designed so that no output is automatically published or acted upon without human review.

**How this system addresses it:**
- All generated descriptions are presented to the host for review
- The host manually copies, edits, and publishes any content they choose to use
- No direct integration with the Airbnb platform exists — the host is always the decision-maker

### 3.3 Data and Data Governance
The system uses publicly available, non-personal data from Inside Airbnb.

**How this system addresses it:**
- Input dataset: Inside Airbnb Berlin listings CSV — publicly available
- No personal data of hosts or guests is processed in the ranking or generation steps
- User-submitted listing details (price, amenities) are processed transiently and not stored beyond the session unless the user opts into logging

## 4. Conformity Assessment Summary

### System Description
The Berlin Airbnb Competitive Intelligence & Listing Optimisation Tool is an AI-powered analytics tool designed for independent Airbnb hosts. It analyses a host's listing against comparable properties in the Berlin market, generates a competitive ranking, identifies key differentiators, and produces an optimised listing description using a large language model (GPT-4o-mini via OpenAI API).

### Risk Class
**LIMITED RISK** under the EU AI Act (Regulation (EU) 2024/1689)

### Basis for Classification
The system generates AI-produced text content (listing descriptions and business insights) that users may choose to publish on third-party platforms. This triggers the transparency obligations under Article 50. The system does not operate in any high-risk domain as defined in Annex III of the EU AI Act, does not make autonomous decisions affecting individuals' rights or access to services, and does not interact with vulnerable populations.

### Applicable Obligations and Design Response

| Obligation | Article | Design Response |
|------------|---------|-----------------|
| Disclose AI-generated content | Art. 50(1) | UI label on all generated outputs |
| Human oversight | Art. 14 (best practice) | No auto-publish; host reviews all output |
| Data governance | Art. 10 (best practice) | Public dataset only; no personal data in model inputs |
| Logging and traceability | Art. 12 (best practice) | LangSmith traces all AI calls |

### Identified Gaps and Resolution Plan

| Gap | Resolution Before Deployment |
|-----|------------------------------|
| No formal AI disclosure policy document | Draft and publish a Terms of Use stating AI content generation |
| No user consent mechanism for LangSmith logging | Add opt-in/opt-out toggle in UI before production launch |
| OpenAI data processing agreement not yet formalised | Execute DPA with OpenAI before handling any user-submitted data at scale |
